SQL queries written against a relational database containing movie, actor, address, and payment tables.

---

## Overview

This project demonstrates SQL skills ranging from basic aggregation to subqueries and date functions, with a focus on writing clean, readable queries that answer real business questions.

---

## Queries

| # | Topic | Concepts Used |
|---|-------|--------------|
| 1 | Actor last name frequency | GROUP BY, COUNT, ORDER BY |
| 2 | Shared last names (3+ actors) | HAVING, DESC sort |
| 3 | Comedy films by revenue | WHERE with LIKE, ORDER BY |
| 4 | Films per year | GROUP BY, COUNT |
| 5 | Prolific directors (4+ films) | HAVING, multi-column sort |
| 6 | Highest revenue per year | MAX, GROUP BY |
| 7 | Postal codes by district letter | LEFT(), IN(), MAX |
| 8 | Top 15 customers by avg spend | ROUND, HAVING, LIMIT |
| 9 | Actors sharing last names | Subquery with IN |
| 10 | Monthly revenue | EXTRACT, SUM |
